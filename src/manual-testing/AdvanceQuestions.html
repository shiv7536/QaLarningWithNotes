API Testing

What is API Testing?
API testing is the process of testing backend services without UI to verify functionality, reliability, performance, and security.
What is REST API?
REST (Representational State Transfer) is an architectural style that uses HTTP methods like GET, POST, PUT, DELETE.
What are HTTP methods?
GET (fetch data), POST (create data), PUT (update full data), PATCH (partial update), DELETE (remove data).
What is Status Code 200?
It means the request was successful.
What is Status Code 201?
It means resource is successfully created.
What is Status Code 400?
Bad request – client side error.
What is Status Code 401 & 403?
401 = Unauthorized, 403 = Forbidden.
What is Status Code 500?
Internal server error.
What is JSON?
JavaScript Object Notation – lightweight data format used in APIs.
What is Request and Response?
Request is sent to server, Response is returned from server.
What is Header in API?
Metadata sent with request like Authorization, Content-Type.
What is Authentication?
Process of verifying user identity (API key, token, OAuth).
What is Authorization?
Checking user permissions after authentication.
What is Postman?
A popular tool used for API testing.
What is Swagger?
Tool for API documentation and testing.
What is API Testing vs UI Testing?
API testing is faster and checks backend logic; UI testing checks frontend behavior.
What will you test in API?
Status codes, response body, headers, response time, security, data validation.
What is Response Time?
Time taken by server to respond to a request.
What is Rate Limiting?
Limiting number of API requests per user/time.
What is Idempotent API?
Multiple identical requests give same result (e.g., GET, PUT).

Selenium

What is Selenium?
Selenium is an open-source tool used for automating web browsers.
What is WebDriver?
WebDriver is an interface that allows interaction with browsers.
Difference between findElement and findElements?
findElement returns single element and throws exception if not found; findElements returns list and returns empty list if not found.
What are locators?
Ways to identify elements like ID, Name, XPath, CSS Selector.
What is XPath?
XPath is a locator used to find elements in XML/HTML structure.
What is implicit wait?
Wait applied globally for element search.
What is explicit wait?
Wait for specific condition using WebDriverWait.
What is Page Object Model (POM)?
Design pattern to separate page elements and test logic.
How to handle alerts?
Using driver.switchTo().alert().accept() or dismiss().
How to handle dropdown?
Using Select class (selectByVisibleText, selectByValue).

Selenium + Maven + TestNG

What is Maven?
Maven is a build automation tool used to manage dependencies and project lifecycle.
What is pom.xml?
It is a configuration file in Maven that contains dependencies, plugins, and project information.
What is dependency in Maven?
External libraries required for project execution (e.g., Selenium, TestNG).
What is TestNG?
TestNG is a testing framework used for test execution, grouping, parallel testing, and reporting.
What are TestNG annotations?
@Test, @BeforeSuite, @AfterSuite, @BeforeMethod, @AfterMethod, etc.
Difference between @BeforeMethod and @BeforeClass?
@BeforeMethod runs before each test method, @BeforeClass runs once before all methods in class.
What is testng.xml?
It is used to configure test execution, grouping, and parallel runs.
How to run tests using Maven?
Using command: mvn test
What is grouping in TestNG?
Grouping allows running specific sets of test cases using group names.
What is parallel testing in TestNG?
Running multiple tests simultaneously to reduce execution time.
What is DataProvider in TestNG?
Used to run test methods with multiple sets of data.
What is Assert in TestNG?
Used to validate expected vs actual results.
What is SoftAssert?
Allows multiple validations without stopping test execution.
What is Page Object Model (POM)?
Design pattern to separate UI elements and test logic.
How to handle synchronization issues in Selenium?
Using waits like implicit, explicit, and fluent wait.
What is Fluent Wait?
Wait with polling frequency and exception handling.
How to take screenshot in Selenium?
Using TakesScreenshot interface.
How to handle frames in Selenium?
Using driver.switchTo().frame()
How to handle multiple windows?
Using getWindowHandles() and switchTo().window()
What is Thread.sleep vs Wait?
Thread.sleep is static wait, while explicit wait is dynamic and better.
How to integrate Selenium with TestNG?
Add TestNG dependency in Maven and use annotations for execution.
How to generate reports in TestNG?
TestNG automatically generates HTML reports after execution.
What is priority in TestNG?
Defines execution order of test methods.
What is dependsOnMethods?
Used to define dependency between test methods.

Scenario Based Questions

Login fail ho raha hai – kaise debug karoge?
1. Check karo credentials sahi hai ya nahi
2. API response check karo (login API fail to nahi ho raha)
3. UI locator change to nahi hua (element mil raha hai ya nahi)
4. Console logs / error logs check karo
5. Network tab me request/response verify karo
6. Environment issue (dev/stage/prod mismatch)
7. Recent changes / deployment check karo
Agar test case pass ho raha hai but production me bug aa gaya to?
1. Test coverage insufficient ho sakta hai
2. Edge cases miss ho gaye
3. Environment difference
4. Data mismatch
5. Reproduce karke root cause find karo
6. Test cases update karo
Page load slow hai – kaise test karoge?
1. Network speed check karo
2. API response time check karo
3. Browser dev tools use karo
4. Performance testing tools (JMeter etc.)
5. Large images / scripts identify karo
Element click nahi ho raha – kya karoge?
1. Check element visible hai ya nahi
2. Wait apply karo (explicit wait)
3. Scroll karo element tak
4. Overlay / popup block to nahi kar raha
5. JavaScript click try karo
Automation test flaky hai (kabhi pass kabhi fail) – kaise fix karoge?
1. Proper waits use karo
2. Stable locators use karo
3. Hardcoded sleep avoid karo
4. Test data fix karo
5. Environment stability check karo
API 200 de rahi hai but data galat hai – kya karoge?
1. Response body validate karo
2. Database verify karo
3. Business logic issue ho sakta hai
4. Backend logs check karo
5. Proper assertions add karo
Checkout flow fail ho raha hai – kaise approach karoge?
1. Step by step flow test karo
2. Payment API check karo
3. Cart data verify karo
4. Logs check karo
5. Edge cases test karo (empty cart, invalid data)
New feature aaya hai – kaise test karoge?
1. Requirement samjho
2. Test cases likho
3. Positive & negative testing karo
4. Regression run karo
5. Edge cases cover karo
App crash ho raha hai – kya steps loge?
1. Logs check karo (Crash logs)
2. Reproduce steps identify karo
3. Device/environment check karo
4. Recent changes verify karo
5. Dev team ke sath share karo
Agar requirement clear nahi hai to kya karoge?
1. BA/PO se clarification lo
2. Assumptions document karo
3. Team discussion karo
4. Test cases finalize karo

Q3: How you handled API testing?
I used Postman for API testing. I validated: - Status codes - Response body - Headers - Response time For example, I tested login API and verified token generation and error responses for invalid credentials.
