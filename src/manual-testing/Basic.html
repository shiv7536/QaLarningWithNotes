Q1: What is a Test Case?

A Test Case is a step-by-step procedure used to verify whether a specific feature of a software application works as expected.

Components of a Test Case:

  • Test Case ID: Unique identifier
  • Title/Description: Purpose of the test
  • Preconditions: Required setup before execution
  • Test Steps: Actions to perform
  • Test Data: Input data for testing
  • Expected Result: Expected outcome
  • Actual Result: Actual outcome after execution
  • Status: Pass/Fail
  • Postconditions: State after test execution

Q2: Why are Test Cases Important?

Test cases ensure that software functions correctly and meets requirements. They help identify bugs early, maintain quality, and provide a structured approach to testing.

Q3: White Box Testing

White Box Testing involves testing the internal structure and logic of an application. Testers have access to the source code and design test cases based on it.

Q4: Black Box Testing

Black Box Testing focuses on validating functionality without knowing the internal implementation. Testing is based on requirements and expected outputs.
